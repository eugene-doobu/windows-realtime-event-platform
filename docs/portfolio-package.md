# Public Repo Package

## What The Public Repo Must Show

- clear Python and C++ split
- large-scale agent simulation as the core technical problem
- reproducible benchmark runs
- validation as a built-in system feature
- clean separation between engine and scenario content

## What The Public Repo Must Not Contain

- real Korean policy scenario files
- real stakeholder packs
- real intervention playbooks
- sensitive source documents
- hardcoded domain-specific business logic

## Public Demo Shape

Use one synthetic benchmark scenario only.

The public demo should show:

- one scenario file
- one run command or API call
- artifacts written by the engine
- validation output
- concise report

## Resume And Portfolio Value

This repo should make the following claims defensible:

- built a C++ simulation kernel for large-scale agent processing
- integrated Python control plane with a native compute core
- designed reproducible scenario execution
- implemented behavioral validation for agent-based simulation
- kept public code domain-neutral while supporting private scenario injection
